# 长跑记录

FirmenABC 这种目录站，全量采集的难点通常在长跑过程。

## 拆城市跑

按城市/区拆分有几个好处：

- 某个地区失败时，不影响其他地区。
- 统计更清楚。
- 方便中断续跑。
- 交付时可以按地区筛选。

## 分页覆盖

列表页要一直翻到没有下一页。采集时建议记录：

- 当前地区。
- 已访问列表页数。
- 已发现详情页数量。
- 已成功解析数量。
- 下一页 URL。

## 失败记录

不要静默跳过失败详情页。至少保留：

```text
state, district, source_url, status, note, scraped_at
```

后面可以重试，也可以在交付说明里解释。

## 速度

稳定性排在速度前面。

这个站点存在临时请求失败和频率限制风险。遇到 429、超时、网络错误时，处理顺序通常是降速、重试、记录，确认状态稳定后考虑并发。
